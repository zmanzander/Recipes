# Instructions for Managing the Recipe Website

The recipe website at `zmanzander/Recipes` uses a **JSON + build script** system. All content is driven by JSON data files. You **never edit HTML files directly** — you edit JSON, then run `node build.js` to regenerate all HTML.

## Repository Structure

```
data/
  weeks.json          <- Weekly meal plan schedules (all weeks)
  ratings.json        <- Recipe ratings ("loved", "good", "skip")
  recipes/
    recipe-id.json    <- One file per recipe (metadata + HTML body)
build.js              <- Generates all HTML from JSON (DO NOT EDIT)
extract.js            <- One-time migration tool (DO NOT USE)
recipes/              <- Generated HTML (DO NOT EDIT DIRECTLY)
catalog.html          <- Generated (DO NOT EDIT DIRECTLY)
index.html            <- Generated (DO NOT EDIT DIRECTLY)
weeks/                <- Generated (DO NOT EDIT DIRECTLY)
```

## CRITICAL RULES

1. **Never edit files in `recipes/`, `weeks/`, `catalog.html`, or `index.html` directly.** They are generated by `build.js` and will be overwritten.
2. **Always run `node build.js` after any JSON change** before committing.
3. **Always commit and push both the JSON changes AND the generated HTML.**
4. **Never edit or run `extract.js`** — it was a one-time migration tool.

## TASK 1: Adding a New Week's Meal Plan

### Step 1: Create recipe JSON files

For each **new** recipe (not already in `data/recipes/`), create `data/recipes/<recipe-id>.json`:

```json
{
  "id": "recipe-id",
  "title": "Human-Readable Recipe Title",
  "protein": "chicken",
  "diet": ["gf", "df", "ef", "sf"],
  "time": "Prep: 15 min | Cook: 30 min",
  "style": "Sheet Pan",
  "rating": null,
  "headStyle": "<CSS for this recipe's page>",
  "bodyHtml": "<HTML content of the recipe body>"
}
```

**Field reference:**

- `id`: Filename without `.json`, lowercase with hyphens (e.g. `lemon-herb-chicken`)
- `protein`: One of `"chicken"`, `"beef"`, `"turkey"`, `"pork"`, `"seafood"`, `"breakfast"`
- `diet`: Array of codes — `"gf"` (Gluten-Free), `"df"` (Dairy-Free), `"ef"` (Egg-Free), `"sf"` (Soy-Free), `"lf"` (Low-FODMAP). Include all that apply. If none, use `[]`
- `time`: Formatted string like `"Prep: 10 min | Cook: 25 min | Total: 35 min"` or `null`
- `style`: Cooking style like `"Sheet Pan"`, `"One Pot"`, `"Fresh Cook"`, `"Dinner Party Fresh Cook"`, or `null`
- `rating`: Set to `null` for new recipes. Ratings are managed in `data/ratings.json`
- `headStyle`: The full CSS that styles this recipe's page. You can reuse this default template or create custom styles:

```css
body { font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Helvetica, Arial, sans-serif; line-height: 1.6; max-width: 800px; margin: auto; padding: 20px; color: #333; }
h1 { color: #2c3e50; border-bottom: 2px solid #3498db; padding-bottom: 10px; }
.meta { background: #ecf0f1; padding: 15px; border-radius: 5px; margin: 20px 0; }
.meta span { margin-right: 20px; }
h2 { color: #34495e; margin-top: 30px; }
ul { padding-left: 20px; }
li { margin: 8px 0; }
ol li { margin: 15px 0; }
.portions { background: #e8f6f3; padding: 15px; border-radius: 5px; margin: 20px 0; }
.portions h3 { margin-top: 0; color: #16a085; }
.notes { background: #fef9e7; padding: 15px; border-radius: 5px; margin: 20px 0; }
a { color: #3498db; }
```

- `bodyHtml`: The recipe HTML content. **Do NOT include** `<h1>` (the build script adds it), nav bar, dietary badges, or footer. **Do include** the meta div, ingredients, instructions, portions, and notes. Start directly with the `<div class="meta">` or similar content.

### Step 2: Add the week entry to `data/weeks.json`

Append a new object to the JSON array:

```json
{
  "week": 6,
  "dates": "March 23-29, 2026",
  "theme": "Short description of the week",
  "note": "Optional note shown on the week page, or null",
  "targets": "Alexander 2200 cal / 200g protein | Hanna 1900 cal / 130g protein",
  "prep": {
    "breakfast": { "recipe": "breakfast-hash", "portions": 10, "note": "Description of breakfast prep" },
    "lunch": { "recipe": "some-lunch-prep", "portions": 6, "note": "Description of lunch prep" }
  },
  "schedule": {
    "sunday": {
      "dinner": { "recipe": "recipe-id", "tag": "fresh", "who": ["zander", "hanna"] }
    },
    "monday": {
      "breakfast": { "recipe": "breakfast-hash", "tag": "prep", "who": ["zander", "hanna"] },
      "lunch": { "recipe": "some-lunch-prep", "tag": "prep", "who": ["zander", "hanna"] },
      "dinner": { "recipe": "another-recipe", "tag": "fresh", "who": ["zander", "hanna"] }
    },
    "tuesday": {
      "breakfast": { "recipe": "breakfast-hash", "tag": "prep", "who": ["zander", "hanna"] },
      "lunch": { "recipe": "some-lunch-prep", "tag": "prep", "who": ["zander", "hanna"] },
      "dinner": { "recipe": "dinner-recipe", "tag": "fresh", "who": ["zander", "hanna"] }
    }
  },
  "recipes": ["breakfast-hash", "some-lunch-prep", "recipe-id", "another-recipe", "dinner-recipe"],
  "highlights": ["Highlight Recipe 1", "Highlight Recipe 2", "Highlight Recipe 3"]
}
```

**Schedule field reference:**

- Days: `"sunday"`, `"monday"`, `"tuesday"`, `"wednesday"`, `"thursday"`, `"friday"`, `"saturday"`
- Meals: `"breakfast"`, `"lunch"`, `"dinner"`
- Each meal object fields:
  - `recipe`: The recipe ID (must match a file in `data/recipes/`)
  - `tag`: `"prep"` (meal-prepped), `"fresh"` (cooked that day), or `"guest"` (guest dinner)
  - `who`: Array of `"zander"` and/or `"hanna"`
  - `soloNote` (optional): Text like `"Hanna Solo"` shown on the schedule
  - `guestNote` (optional): Custom text for guest tag (e.g. `"Guest + 1"`)
- Special day entries:
  - Free day: `"saturday": { "note": "Free Day" }`
  - Text-only meal: `"lunch": { "label": "Leftovers" }` or `"dinner": { "label": "Dinner Out" }`
- `prep` (optional): Describes batch prep at the top level — `recipe`, `portions`, `note`
- `recipes`: Array of **all** recipe IDs used in this week (used for catalog week tags)
- `highlights`: Array of 3-4 recipe title strings shown on the weekly archives index cards

### Step 3: Build and push

```bash
node build.js
git add data/ recipes/ catalog.html index.html weeks/
git commit -m "Add Week 6 meal plan: [theme description]"
git push origin main
```

## TASK 2: Editing a Recipe Mid-Week

When Zander or Hanna requests changes to an existing recipe (ingredients, portions, steps, etc.):

### Step 1: Edit the recipe JSON

Open `data/recipes/<recipe-id>.json` and modify the `bodyHtml` field. This is the HTML content of the recipe.

**Common edits:**

- **Change an ingredient amount**: Find the ingredient in the `bodyHtml` HTML and update the quantity. For example, change `1.5 lbs ground turkey` to `2 lbs ground turkey`.
- **Add an ingredient**: Add a new `<li>` to the ingredients `<ul>` list in `bodyHtml`.
- **Remove an ingredient**: Delete the `<li>` from the ingredients list.
- **Change portion sizes**: Find the portions div in `bodyHtml` and update the amounts.
- **Change cooking instructions**: Edit the `<ol>` or `<div class="step">` elements in `bodyHtml`.
- **Change prep/cook time**: Update both the `"time"` field AND the time shown in the `bodyHtml` meta div.
- **Change dietary tags**: Update the `"diet"` array. For example, if a recipe is no longer dairy-free after swapping an ingredient, remove `"df"` from the array.

### Step 2: Build and push

```bash
node build.js
git add data/recipes/<recipe-id>.json recipes/<recipe-id>.html catalog.html
git commit -m "Update <recipe-name>: <what changed>"
git push origin main
```

**Important**: The `bodyHtml` contains raw HTML. Be careful with quotes — use `\"` for double quotes inside the JSON string. Maintain proper HTML structure (close all tags).

## TASK 3: Swapping a Recipe Out of a Week

When a recipe should be removed from a week and optionally replaced:

### Step 1: Edit `data/weeks.json`

1. Find the week object in the array.
2. In the `schedule`, find every day/meal that references the old recipe ID and either:
   - **Replace it** with a different recipe ID: `"recipe": "new-recipe-id"`
   - **Remove the meal entirely** by deleting that meal key from the day
   - **Replace with text**: `"dinner": { "label": "Dinner Out" }`
3. Update the `"recipes"` array: remove the old recipe ID, add the new one (if replacing).
4. Update the `"highlights"` array if the swapped recipe was a highlight.

### Step 2: If the replacement is a brand-new recipe

Create `data/recipes/<new-recipe-id>.json` following the format in Task 1, Step 1.

### Step 3: Build and push

```bash
node build.js
git add data/ recipes/ catalog.html index.html weeks/
git commit -m "Week N: swap <old-recipe> for <new-recipe>"
git push origin main
```

**Note**: You do NOT need to delete the old recipe's JSON file from `data/recipes/`. It stays in the catalog as an available recipe for future weeks. Only delete a recipe JSON if it should be permanently removed from the entire site.

---

## TASK 4: Removing a Recipe Permanently

If a recipe should be completely removed from the site:

1. Delete `data/recipes/<recipe-id>.json`
2. Remove the recipe ID from the `"recipes"` array of any week in `data/weeks.json` that references it
3. Remove it from any `"schedule"` entries in those weeks
4. Remove it from `data/ratings.json` if it has a rating
5. Run `node build.js`
6. Delete the generated `recipes/<recipe-id>.html`
7. Commit and push all changes

## TASK 5: Adding a Rating

Edit `data/ratings.json`:

```json
{
  "lemon-herb-chicken": "loved",
  "beef-stir-fry": "good",
  "garlic-butter-shrimp": "skip",
  "new-recipe-id": "loved"
}
```

Valid ratings: `"loved"`, `"good"`, `"skip"`. Then run `node build.js` and push.

## Quick Reference: Recipe IDs Currently in the System

Run `ls data/recipes/` to see all available recipe IDs. Each filename without `.json` is the recipe ID.

## Quick Reference: What `node build.js` Generates

- `recipes/*.html` — All recipe pages (with nav bar, dietary badges, footer)
- `catalog.html` — Searchable/filterable recipe catalog
- `index.html` — Current (latest) week's meal plan
- `weeks/week{N}.html` — Archived week pages (all except current)
- `weeks/index.html` — Weekly archives index

The **current week** is always the last (highest numbered) week in `data/weeks.json`. When you add Week 6, Week 5 automatically becomes archived.
